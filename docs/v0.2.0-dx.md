# NakedCanvas — v0.2.0 "Developer Experience" Spec

> Status: **Draft / In Planning**
> Owner: PO + Engineering
> Milestone: `v0.2.0`
> Depends on: `v0.1.0` (Core MVP) — see [`v0.1.0-core-mvp.md`](./v0.1.0-core-mvp.md)
> Theme: **Make it easy to drop into a real project** — copy-paste friendly, or install as a package.

---

## 1. Objective

v0.1.0 proved the engine runs. v0.2.0 makes it **something a developer actually wants to use**:

1. Physics is **tunable from the outside** — friction, elasticity, mouse radius — without touching internals.
2. The engine **survives window resizing** correctly at any DPR, staying feather-light.
3. It **ships as a package** — clean ESM/UMD targets for private or public npm distribution.

No new visual capability. This milestone is about **ergonomics, robustness, and distribution**.

---

## 2. Baseline — what v0.1.0 already delivered

To avoid re-planning solved work, v0.2.0 builds on these existing foundations:

- `NakedOptions` interface with `friction`, `returnEase`, `mouseRadius`, `pushForce`, `particleGap`, `defaultColor`; constructor accepts `Partial<NakedOptions>` merged over defaults.
- DPR-aware `resize()` and an offscreen sampling buffer (device-pixel correct).
- Vite lib build already emitting **ESM + UMD + `.d.ts`**, `dependencies: {}`, gzip ≈ 1.5KB.

v0.2.0 **hardens and formalizes** these — it does not rebuild them from scratch.

## 3. Scope

### 3.1 In Scope

| #   | Capability | Notes |
| --- | ---------- | ----- |
| D1  | **Live physics customization** via `setOptions(partial)` | Tune coefficients at runtime, not just at construction |
| D2  | **Input validation / clamping** of physics coefficients | Keep `returnEase`, `friction`, etc. in safe ranges |
| D3  | **Lightweight DPR-aware resize module** that re-samples on window change | Debounced; targets recomputed so text stays centered/scaled |
| D4  | **Lifecycle teardown** `destroy()` | Remove listeners + cancel RAF (required to embed in SPAs) |
| D5  | **Publish-ready package** | `exports` map, `files`, metadata, README usage, provenance of ESM/UMD/types |

### 3.2 Out of Scope (deferred to v0.3.0+)

- Per-particle color / gradients / theming.
- Multiple text layers, image sources, or animated transitions between strings.
- Framework wrappers (React/Vue/Svelte).
- Config presets / theme packs.

## 4. Design

### 4.1 Live physics customization (D1, D2)

Expose a single mutation entry point that overwrites option **properties in place** (no reassignment, per code convention):

```ts
public applyOptions(partial: Partial<NakedOptions>): void;
```

- Merges `partial` over the live `options` object by mutating fields, so the running RAF loop immediately reads new values — no restart, no re-instantiation.
- **Clamping (D2):** guard against values that break the simulation:
  - `friction`      → clamp to `[0, 0.99]` (>= 1 never settles)
  - `returnEase`    → clamp to `[0.001, 0.5]`
  - `mouseRadius`   → `>= 0`
  - `pushForce`     → finite number
  - `particleGap`   → `>= 1` (0 would divide/step to infinity)
- Changing `particleGap` or `defaultColor` alone does **not** re-sample; the caller re-invokes `renderText()` if resampling is desired. (Documented behavior.)

> Naming follows the active-voice convention (`applyOptions`, not `setOptions`/`update`).

### 4.2 Lightweight DPR-aware resize module (D3)

Current gap: `resize()` fixes the canvas geometry but the particle **targets (`tx`,`ty`) go stale** — after a resize the text is no longer centered/scaled.

Design:

- Cache the last rendered string (`lastText`) inside `renderText()`.
- On `resize`, after recomputing width/height/DPR, **re-sample** `lastText` so targets track the new geometry.
- **Debounce** the resize handler (trailing, ~100ms) so a drag-resize does not fire hundreds of `getImageData` passes. Debounce implemented from scratch (zero deps) using a stored timer id, cleared/reset in place.
- Keep it allocation-light: the debounce closure is created **once** in `initEvents`, not per event.

### 4.3 Lifecycle teardown (D4)

To embed cleanly in SPAs (component unmount), add:

```ts
public destroy(): void;
```

- Cancels the pending `requestAnimationFrame` (store the RAF handle).
- Removes every listener registered in `initEvents` (resize, mouse*, touch*).
- Requires listener references to be **stable** (bound once), so `initEvents` must store them rather than passing inline arrows.
- After `destroy()`, the instance is inert; calling `renderText()` again is undefined behavior (documented) — create a new instance.

### 4.4 Publish-ready package (D5)

- `package.json`: `name`, `version`, `description`, `keywords`, `repository`, `homepage`, `license`, `author`, `sideEffects: false`, `type: module`.
- `exports` map already present — verify `types`/`import`/`require` all resolve against `dist/`.
- `files: ["dist"]` so only build artifacts publish (no `src`, no configs).
- `dependencies` **must stay `{}`**; build tools live in `devDependencies` only.
- Build order `vite build && tsc` (types survive `emptyOutDir`).
- README: install + minimal usage snippet (both ESM import and UMD `<script>`).
- Confirm UMD global name is `NakedCanvas`.
- Dry-run `npm pack` (via pnpm) and inspect the tarball contents before any publish.

## 5. Public API (v0.2.0 additions)

```ts
class NakedCanvas {
  constructor(canvas: HTMLCanvasElement, options?: Partial<NakedOptions>);
  renderText(text: string): void;

  // --- new in v0.2.0 ---
  applyOptions(partial: Partial<NakedOptions>): void; // live physics tuning (clamped)
  destroy(): void;                                    // remove listeners + stop RAF
}
```

`renderText` gains an internal side effect (caches `lastText` for resize re-sampling) but its signature is unchanged.

## 6. Acceptance Criteria

| ID | Criterion | Measurement |
|----|-----------|-------------|
| AC1 | `applyOptions({ friction, returnEase, mouseRadius, pushForce })` changes behavior live without re-instantiating | Manual: tweak, observe motion change mid-run |
| AC2 | Out-of-range coefficients are clamped, never NaN/frozen simulation | Unit-ish check: pass `friction: 5`, particles still settle |
| AC3 | Resizing the window keeps text centered, correctly scaled, at dpr 1 and 2 | Visual on Retina + external monitor |
| AC4 | Rapid drag-resize does **not** cause jank/GC spikes (debounced) | DevTools Performance during resize drag |
| AC5 | `destroy()` removes all listeners and cancels RAF (no leaks, no ghost loop) | `getEventListeners` / heap check; CPU drops to 0 after destroy |
| AC6 | Package builds ESM + UMD + types; `pnpm pack` tarball contains only `dist/` | `pnpm pack` + tarball inspection |
| AC7 | `dependencies` is still `{}`; gzip bundle still **≤ 2KB** | `package.json` + gzip measure |
| AC8 | Code convention held: no `let`, no `any`, active-voice names, RAF no-`new` | Static grep + review |

---

## 7. Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|-----------|
| Re-sampling on every resize event triggers repeated `getImageData` (jank) | High | Debounce trailing (D3) |
| DX additions (`applyOptions`/`destroy`) push bundle over 2KB | Medium | Keep methods tiny, no helpers; re-measure gzip in AC7 |
| Listener leak defeats `destroy()` because arrows aren't stored | Medium | Store bound handler refs in `initEvents` (D4) |
| Clamping hides genuine user error silently | Low | Clamp + document ranges in README/JSDoc |
| Publishing `src`/configs by accident | Low | `files: ["dist"]` + `pnpm pack` dry-run (AC6) |
